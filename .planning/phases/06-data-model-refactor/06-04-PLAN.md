# Plan: Frontend Updates

## Objective

Update frontend to support the new Work → MediaItem structure with multi-file display and management.

## Requirements

- **DATA-05**: 前端界面支持作品包含多个媒体项的展示
- **DATA-06**: 管理员可以在作品详情中管理媒体项

## Tasks

### Task 1: Create Media Item API Client

Create `frontend/src/api/mediaItems.ts`:

```typescript
export const mediaItemsApi = {
  async addMediaItem(workId: string, data: FormData),
  async getMediaItems(workId: string),
  async updateMediaItem(id: string, data),
  async deleteMediaItem(id: string),
  async reorderMediaItems(workId: string, itemIds: string[]),
}
```

### Task 2: Update Work Types and Store

Update `frontend/src/types/work.ts`:

- Add `MediaItem` interface
- Update `Work` interface to include `mediaItems: MediaItem[]`

Update `frontend/src/stores/works.ts`:

- Handle media items in work operations
- Add actions for media item management

### Task 3: Create Media Item Manager Component

Create `frontend/src/components/MediaItemManager.vue`:

- Display all media items in a work (thumbnail grid)
- Drag to reorder
- Delete individual items
- Upload new items button

### Task 4: Update Works Management Page

Update `frontend/src/views/admin/Works.vue`:

- Show first media item as thumbnail
- Click to see all media items
- Support creating work with multiple files

### Task 5: Update Public Gallery

Update `frontend/src/views/Home.vue` and related components:

- Display works with first media item as cover
- Lightbox shows all media items in a work
- Navigate between media items within a work

### Task 6: Update Share Page

Update `frontend/src/views/Share.vue`:

- Show all media items for shared works
- Download individual media items or all
