# Plan: Backend Service Updates

## Objective

Update backend services to support the new Work → MediaItem structure.

## Requirements

- **DATA-01**: 作品可以包含多个媒体项
- **DATA-03**: 管理员可以为作品添加/删除媒体项

## Tasks

### Task 1: Create MediaItem Service

Create `backend/src/services/mediaItemService.ts`:

```typescript
export class MediaItemService {
  // Create media item for a work
  async createMediaItem(workId: string, data: CreateMediaItemData): Promise<MediaItem>;
  
  // Get all media items for a work
  async getMediaItemsByWork(workId: string): Promise<MediaItem[]>;
  
  // Update media item (position, etc.)
  async updateMediaItem(id: string, data: UpdateMediaItemData): Promise<MediaItem | null>;
  
  // Delete media item
  async deleteMediaItem(id: string): Promise<boolean>;
  
  // Reorder media items
  async reorderMediaItems(workId: string, itemIds: string[]): Promise<void>;
}
```

### Task 2: Update Work Service

Update `backend/src/services/workService.ts`:

- `createWork`: Accept optional `mediaItems` array or create empty work
- `getWorkById`: Include `mediaItems` relation
- `getWorks`: Include `mediaItems` relation
- `deleteWork`: Delete all associated media items first

### Task 3: Update Upload Service

Update `backend/src/services/uploadService.ts`:

- Add `uploadMultipleFiles` method for batch uploads
- Process each file (thumbnails, etc.) and return array of results

### Task 4: Update Public Service

Update `backend/src/services/publicService.ts`:

- `getPublicWorks`: Include `mediaItems` relation, use first item as cover
- `getPublicWorkById`: Include `mediaItems` relation
- Album cover: Use first media item of first work
