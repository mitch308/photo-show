# Plan: Backend API Updates

## Objective

Update backend API routes to support the new Work → MediaItem structure.

## Requirements

- **DATA-01**: 作品可以包含多个媒体项
- **DATA-03**: 管理员可以为作品添加/删除媒体项
- **DATA-04**: API 提供媒体项管理接口

## Tasks

### Task 1: Create Media Item Routes

Create `backend/src/routes/mediaItems.ts`:

```
POST   /api/works/:workId/media       - Add media item to work
GET    /api/works/:workId/media       - Get all media items for work
PUT    /api/media/:id                 - Update media item
DELETE /api/media/:id                 - Delete media item
POST   /api/works/:workId/media/reorder - Reorder media items
```

### Task 2: Update Work Routes

Update `backend/src/routes/works.ts`:

- `POST /api/works`: Create work (optionally with media items)
- `GET /api/works`: Return works with media items
- `GET /api/works/:id`: Return work with media items
- `PUT /api/works/:id`: Update work metadata
- `DELETE /api/works/:id`: Delete work and all media items

### Task 3: Update Upload Routes

Update `backend/src/routes/upload.ts`:

- `POST /api/upload`: Single file upload (backward compatible)
- `POST /api/upload/multiple`: Multiple file upload
- Return format includes workId for linking

### Task 4: Update Public Routes

Update `backend/src/routes/public.ts`:

- `GET /api/public/works`: Return works with first media item as thumbnail
- `GET /api/public/works/:id`: Return work with all media items
- `GET /api/public/albums/:id`: Return album with works containing media items

### Task 5: Update Share Routes

Update `backend/src/routes/share.ts`:

- Share page shows all media items for shared works
- Download handles multiple media items

## Files Modified
