# Plan: Create MediaItem Model & Update Work Model

## Objective

Create new MediaItem model to support multiple media files per work, and update Work model to be a logical container.

## Requirements

- **DATA-01**: 作品可以包含多个媒体项（图片或视频）
- **DATA-02**: 数据模型支持 Album → Work → MediaItem 三层结构

## Tasks

### Task 1: Create MediaItem Model

Create `backend/src/models/MediaItem.ts`:

```typescript
@Entity('media_items')
export class MediaItem {
  @PrimaryColumn({ type: 'varchar', length: 36 })
  id: string;

  @Column({ type: 'varchar', length: 36 })
  workId: string;

  @ManyToOne(() => Work, work => work.mediaItems)
  @JoinColumn({ name: 'work_id' })
  work: Work;

  @Column({ type: 'varchar', length: 500 })
  filePath: string;

  @Column({ type: 'varchar', length: 500, nullable: true })
  thumbnailSmall: string;

  @Column({ type: 'varchar', length: 500, nullable: true })
  thumbnailLarge: string;

  @Column({ type: 'varchar', length: 255 })
  originalFilename: string;

  @Column({ type: 'varchar', length: 50 })
  fileType: 'image' | 'video';

  @Column({ type: 'varchar', length: 50 })
  mimeType: string;

  @Column({ type: 'int', default: 0 })
  fileSize: number;

  @Column({ type: 'int', default: 0 })
  position: number;

  @CreateDateColumn()
  createdAt: Date;
}
```

### Task 2: Update Work Model

Update `backend/src/models/Work.ts`:

- Remove: `filePath`, `thumbnailSmall`, `thumbnailLarge`, `originalFilename`, `fileType`, `mimeType`, `fileSize`
- Add: `@OneToMany(() => MediaItem, item => item.work)` relationship
- Keep: `title`, `description`, `position`, `isPinned`, `isPublic`, `viewCount`, `downloadCount`, `albums`, `tags`

### Task 3: Create Data Migration Script

Create `backend/src/migrations/001-work-to-media-item.ts`:

For each existing Work:
1. Create a MediaItem with the work's file fields
2. Link MediaItem to the Work
3. Set MediaItem position to 0

### Task 4: Register MediaItem Model

Update `backend/src/models/index.ts` to export MediaItem.
