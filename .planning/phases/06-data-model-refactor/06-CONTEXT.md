# Phase 6: 数据模型重构

**创建时间：** 2026-03-25
**状态：** 未开始

## 背景

当前系统假设一个作品对应一个文件（图片或视频），一个相册包含多个作品。但实际使用中，摄影师经常需要：

1. **一个作品包含多个媒体项**：
   - 例如：一个婚纱摄影作品可能包含 10-20 张精选照片
   - 例如：一个活动记录作品可能包含多个视频片段和照片

2. **相册作为作品的集合**：
   - 相册不再直接包含图片/视频
   - 相册包含作品，作品包含媒体项

## 重构目标

### 数据模型变更

**现有模型：**
```
Album (相册) 1:N Work (作品/文件)
Work = 单个文件 (图片或视频)
```

**目标模型：**
```
Album (相册) 1:N Work (作品)
Work 1:N MediaItem (媒体项)
MediaItem = 单个文件 (图片或视频)
```

### 关键变化

1. **Work 表变化**：
   - 不再直接存储文件路径
   - 成为媒体项的逻辑容器
   - 保留标题、描述、标签等元数据

2. **新增 MediaItem 表**：
   - 存储实际的文件信息
   - 类型字段（图片/视频）
   - 文件路径、缩略图、尺寸等

3. **关系调整**：
   - Album → Work：一对多（不变）
   - Work → MediaItem：一对多（新增）
