# Plan: Backend Enhancement APIs

## Objective

Create backend APIs for batch operations, statistics, client management, and share link enhancements.

## Requirements

- **BATCH-01~04**: 批量选择、修改状态、移动、删除作品
- **STAT-01~04**: 作品/相册浏览下载统计
- **PRIV-06**: 私密链接访问次数限制

## Tasks

### Task 1: Batch Operations Service & Routes

Create `backend/src/services/batchService.ts` and `backend/src/routes/batch.ts`:

- `POST /api/batch/works/status` — 批量修改公开状态
- `POST /api/batch/works/move` — 批量移动到相册
- `POST /api/batch/works/delete` — 批量删除作品

### Task 2: Statistics Service & Routes

Create `backend/src/services/statsService.ts` and `backend/src/routes/stats.ts`:

- `GET /api/stats/works` — 作品统计列表（浏览、下载）
- `GET /api/stats/albums` — 相册统计列表（浏览）
- `GET /api/stats/overview` — 总览数据

### Task 3: Client Service & Routes

Create `backend/src/services/clientService.ts` and `backend/src/routes/clients.ts`:

- `GET /api/clients` — 客户列表
- `POST /api/clients` — 创建客户
- `PUT /api/clients/:id` — 更新客户
- `DELETE /api/clients/:id` — 删除客户
- `GET /api/clients/:id/shares` — 客户关联的分享链接
- `GET /api/clients/:id/access-logs` — 客户访问历史

### Task 4: Share Link Enhancements

Update `backend/src/services/shareService.ts`:

- Add `maxAccess` field to token creation
- Add `clientId` field to token creation
- Track access count in Redis

Update `backend/src/routes/admin/share.ts`:

- Add `maxAccess` and `clientId` to create request
- Return access logs for each share

### Task 5: Access Log Recording

Create `backend/src/services/accessLogService.ts`:

- `recordAccess(token, workId, action, req)` — 记录访问

Update `backend/src/routes/share.ts`:

- Call `recordAccess` on view and download
