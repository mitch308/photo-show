# Plan: Client Model & Access Log Schema

## Objective

Create database models for Client management and ShareAccessLog for tracking private link visits.

## Requirements

- **CLNT-01**: 管理员可以添加和管理客户信息
- **CLNT-04**: 管理员可以编辑和删除客户信息
- **PRIV-05**: 管理员可以查看私密链接的访问记录
- **STAT-04**: 浏览和下载次数在管理后台可见

## Tasks

### Task 1: Create Client Model

Create `backend/src/models/Client.ts`:

```typescript
@Entity('clients')
export class Client {
  @PrimaryColumn({ type: 'varchar', length: 36 })
  id: string;

  @Column({ type: 'varchar', length: 100 })
  name: string;

  @Column({ type: 'varchar', length: 50, nullable: true })
  phone: string;

  @Column({ type: 'varchar', length: 100, nullable: true })
  email: string;

  @Column({ type: 'varchar', length: 100, nullable: true })
  company: string;

  @Column({ type: 'varchar', length: 255, nullable: true })
  address: string;

  @Column({ type: 'date', nullable: true })
  birthday: Date;

  @Column({ type: 'text', nullable: true })
  notes: string;

  @OneToMany(() => ShareToken, token => token.client)
  shareTokens: ShareToken[];

  @CreateDateColumn()
  createdAt: Date;

  @UpdateDateColumn()
  updatedAt: Date;
}
```

### Task 2: Create ShareAccessLog Model

Create `backend/src/models/ShareAccessLog.ts`:

```typescript
@Entity('share_access_logs')
export class ShareAccessLog {
  @PrimaryColumn({ type: 'varchar', length: 36 })
  id: string;

  @Column({ type: 'varchar', length: 64 })
  token: string;

  @Column({ type: 'varchar', length: 36 })
  workId: string;

  @Column({ type: 'varchar', length: 20 })
  action: 'view' | 'download';

  @Column({ type: 'varchar', length: 45, nullable: true })
  ipAddress: string;

  @Column({ type: 'varchar', length: 255, nullable: true })
  userAgent: string;

  @CreateDateColumn()
  createdAt: Date;
}
```

### Task 3: Update ShareToken Model

Add `clientId` and `maxAccess` fields to existing ShareToken storage in Redis, or create MySQL model if needed.

### Task 4: Register Models

Update `backend/src/models/index.ts` to export new models.
