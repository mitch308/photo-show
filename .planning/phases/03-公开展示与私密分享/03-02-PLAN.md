  <action>
    Create `backend/src/services/shareService.ts`:
    
    **Import crypto for token generation:**
    ```typescript
    import crypto from 'crypto';
    import { getRedis } from '../config/redis.js';
    import type { Redis } from 'ioredis';
    ```
    
    **Interface ShareTokenData:**
    ```typescript
    export interface ShareTokenData {
      workIds: string[];
      expiresAt: number;  // Unix timestamp (ms)
      createdAt: number;
      createdBy?: string; // Admin ID (optional)
    }
    ```
    
    **Methods:**
    
    1. `generateToken(): string`
       - Per D-09: `crypto.randomBytes(32).toString('base64url')`
       - Returns ~43 character URL-safe string
    
    2. `createShareToken(workIds: string[], expiresInDays: number = 7): Promise<string>`
       - Generate token using generateToken()
       - Calculate expiresAt = now + expiresInDays * 24 * 60 * 60 * 1000
       - Calculate TTL in seconds = expiresInDays * 24 * 60 * 60
       - Store in Redis: key=`share:{token}`, value=JSON.stringify(data)
       - Use `redis.setex(key, ttl, value)` for auto-expiration (per D-10)
    
    3. `validateToken(token: string): Promise<ShareTokenData | null>`
       - Get from Redis: `redis.get('share:{token}')`
       - Return null if not found (expired or invalid)
       - Parse JSON and return ShareTokenData
    
    4. `revokeToken(token: string): Promise<boolean>`
       - Delete from Redis: `redis.del('share:{token}')`
       - Return true if deleted, false if not found
    
    5. `isWorkInShare(token: string, workId: string): Promise<boolean>`
       - Validate token
       - Check if workId is in workIds array
       - Return true/false
    
    **Expiration options (per D-11):**
    - 1 day: 86400 seconds
    - 7 days: 604800 seconds (default)
    - 30 days: 2592000 seconds
    - Custom: user-provided days
    
    Export singleton instance: `export const shareService = new ShareService();`
  </action>
