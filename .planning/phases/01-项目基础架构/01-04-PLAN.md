## Backend API Contract (from Plan 03)

- `POST /api/auth/login` → { code, data: { admin }, message }
- `POST /api/auth/logout` → { code, data: null, message }
- `POST /api/auth/refresh` → { code, data: { accessToken }, message }
- `GET /api/auth/me` → { code, data: { admin }, message }

All tokens in httpOnly cookies (automatic).

  <name>Task 3: Implement Route Guards</name>
  <files>frontend/src/router/guards.ts, frontend/src/router/index.ts</files>
  <action>
    Create navigation guards to protect admin routes and handle auth state.

    1. Create `src/router/guards.ts`:
       ```typescript
       export function setupRouterGuards(router: Router, authStore: ReturnType<typeof useAuthStore>) {
         router.beforeEach(async (to, from, next) => {
           // Check if route requires auth
           if (to.meta.requiresAuth) {
             if (!authStore.isAuthenticated) {
               // Try to check auth status
               const isAuthed = await authStore.checkAuth();
               if (!isAuthed) {
                 return next({ name: 'Login', query: { redirect: to.fullPath } });
               }
             }
           }

           // Redirect logged-in users away from login page
           if (to.name === 'Login' && authStore.isAuthenticated) {
             return next({ name: 'Admin' });
           }

           next();
         });
       }
       ```

    2. Update `src/router/index.ts`:
       - Add meta.requiresAuth to admin routes
       - Import and setup guards in router creation

    3. Handle token refresh on 401:

       - If 401 and has refresh token cookie
       - Call refresh endpoint
       - Retry original request
