# Plan 09-01: 文件信息展示

**Requirement:** WORK-01 — 作品列表展示文件总大小和文件数量

**Objective:** 在作品列表表格中显示每个作品的文件数量和总文件大小，支持多文件作品（mediaItems）并兼容旧数据。

---

## Context

**Why:** Phase 6 引入了 MediaItem 模型，作品可包含多个文件，但作品列表仍显示 deprecated 的单文件大小。管理员需要看到每个作品的实际文件数量和总大小。

**Key insight:** 后端 `workService.getWorks()` 已返回 `mediaItems` 关联（见 `backend/src/services/workService.ts:118`），前端只需计算聚合值，无需修改后端。

---

## Tasks

### Task 1: 添加文件数量计算逻辑

<read_first>
- frontend/src/views/admin/Works.vue — 当前表格结构
- frontend/src/api/types.ts:56 — Work 类型定义（含 mediaItems 字段）
</read_first>

<action>
在 `frontend/src/views/admin/Works.vue` 的 `<script setup>` 部分，在 `formatFileSize` 函数（约第 124 行）下方添加两个新函数：

```typescript
function getFileCount(work: Work): number {
  return work.mediaItems?.length || 1;
}

function getTotalFileSize(work: Work): number {
  if (work.mediaItems && work.mediaItems.length > 0) {
    return work.mediaItems.reduce((sum, item) => sum + item.fileSize, 0);
  }
  return work.fileSize || 0;
}
```

说明：
- `getFileCount`: 返回 mediaItems 数量，无则返回 1（兼容旧作品）
- `getTotalFileSize`: 聚合 mediaItems 的 fileSize，无则 fallback 到 work.fileSize
</action>

<acceptance_criteria>
- `frontend/src/views/admin/Works.vue` 包含 `function getFileCount(work: Work): number`
- `frontend/src/views/admin/Works.vue` 包含 `function getTotalFileSize(work: Work): number`
- `getTotalFileSize` 函数包含 `work.mediaItems.reduce` 调用
- `getTotalFileSize` 函数包含 fallback `return work.fileSize || 0`
</acceptance_criteria>
