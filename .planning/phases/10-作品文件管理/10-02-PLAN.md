# Plan 10-02: 删除文件功能

**Requirement:** WORK-04 — 管理员可以从已有作品中删除文件

**Objective:** 在编辑作品对话框的文件列表中，为每个文件添加删除按钮，让管理员可以删除不需要的文件。最后一个文件需阻止删除并提示。

---

## Context

**Why:** 作品可能包含错误上传的文件或需要移除的文件，管理员需要删除功能。

**Key insight:** 后端 API 已实现（`DELETE /media/:id`），前端只需：
1. 在文件列表项中添加删除按钮
2. 删除前检查是否为最后一个文件
3. 确认后调用 `mediaItemsApi.deleteMediaItem()`

---

## Tasks

### Task 1: 添加删除文件处理函数

<read_first>
- frontend/src/views/admin/Works.vue — 当前的 handleAddFile 函数（Plan 10-01 添加的）
- frontend/src/api/mediaItems.ts:54 — deleteMediaItem 函数签名
</read_first>

<action>
在 `frontend/src/views/admin/Works.vue` 的 `<script setup>` 部分，在 `cancelAddFile` 函数后添加删除文件处理函数：

```typescript
async function handleDeleteFile(item: MediaItem) {
  if (!editingWork.value) return;
  
  if (editingWork.value.mediaItems && editingWork.value.mediaItems.length <= 1) {
    ElMessage.warning('作品至少需要保留一个文件，请先上传其他文件');
    return;
  }
  
  try {
    await ElMessageBox.confirm(
      `确定要删除文件「${item.originalFilename}」吗？此操作不可恢复。`,
      '删除确认',
      { type: 'warning' }
    );
    
    await mediaItemsApi.deleteMediaItem(item.id);
    ElMessage.success('文件删除成功');
    
    await worksStore.fetchWorks();
    
    const updated = worksStore.works.find(w => w.id === editingWork.value?.id);
    if (updated) {
      editingWork.value = updated;
    }
  } catch {
    // 用户取消
  }
}
```

说明：
- 首先检查是否为最后一个文件，是则阻止删除
- 使用 ElMessageBox.confirm 二次确认
- 删除成功后刷新作品列表并更新编辑中的作品
</action>

- 函数包含 `ElMessage.warning('作品至少需要保留一个文件` 提示
- 函数包含 `await mediaItemsApi.deleteMediaItem(item.id)` 调用
